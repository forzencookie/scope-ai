# K10 — Kvalificerade andelar

Test page: `/test-ui/walkthroughs/k10/`

## What it shows

Scooby calculates the gränsbelopp (boundary amount) for a shareholder under the 3:12 rules and shows how much dividend can be taxed at 20% vs as employment income. Every number traces to its source — aktiebok, löneregister, or prior K10.

## UI Components

- **ScoobyPresentation** — header with message + 3 highlights (gränsbelopp, föreslagen utdelning, total skatt)
- **WalkthroughRenderer** (block-based) with:
  - Ägande section: annotation (source: aktiebok), key-value (delägare, personnummer, aktier, aktieslag, anskaffningsutgift)
  - Alternativ 1: Schablonbelopp — key-value (2.75 × IBB, din andel)
  - Alternativ 2: Lönebaserat utrymme — annotation (source: löneregister), financial-table (all employees with total lön and source), key-value (50% av lönesumman, lönekrav check)
  - Info-card: which method wins
  - Sparat utdelningsutrymme: annotation (source: prior K10 fält 3.18), key-value (sparat, uppräkning, totalt)
  - Slutligt gränsbelopp: key-value (base + saved = total)
  - Skatteuträkning: key-value (20% portion, tjänstedel, skatt, kvar efter skatt)
  - Status-check: aktiebok verifierad, löneunderlag komplett, sparat utrymme hämtat, utdelning inom gräns
  - Action-bar: Exportera K10 (SRU), Stäng

## Expected behavior

When user asks "Hur mycket utdelning kan jag ta?" or "Gör min K10":

1. Scooby identifies the active shareholder (from session/context)
2. Tool fetches: aktiebok data, all salary data for prior year, prior K10 saved space
3. Tool computes gränsbelopp using both schablon and lönebaserat methods
4. Returns a WalkthroughResponse with the full calculation chain

## Data connections — what's dynamic

| Data | Source | Static in test? |
|------|--------|----------------|
| Shareholder name, personnummer, shares | `shareholders` + `share_register` tables | Yes |
| Aktieslag, anskaffningsutgift, acquisition date | `share_register` table | Yes |
| All employee salaries for prior year | `payroll_runs` + `payroll_line_items` | Yes |
| IBB (inkomstbasbelopp) | Yearly government rate — data source | Yes |
| Schablonbelopp (2.75 × IBB) | Computed from IBB | Yes |
| Salary threshold (6 IBB + 5%) | Computed from IBB + total salaries | Yes |
| Prior year saved dividend space | Prior K10 declaration or `tax_declarations` table | Yes |
| Uppräkningsränta (SLR + 3%) | Riksbanken statslåneränta — yearly rate | Yes |
| Proposed dividend amount | User input or `board_minutes` decision | Yes |
| Tax rates (20% kapital, ~52% tjänst) | Tax law — can change | Yes |

## What must NEVER be static in production

- **Share register data** — from aktiebok, changes with transactions
- **Salary data** — from actual payroll runs per year
- **IBB** — changes yearly (government published rate)
- **Uppräkningsränta** — changes yearly (Riksbanken SLR)
- **Saved dividend space** — from prior K10 or tax filing
- **Proposed dividend** — user decision or board minutes

## What CAN be static

- Field codes ("fält 1.3", "fält 1.5", etc.)
- Legal references ("IL 57 kap — 3:12-reglerna")
- Calculation method names ("Schablonbelopp", "Lönebaserat utrymme")
- Explanation text for salary threshold rule
